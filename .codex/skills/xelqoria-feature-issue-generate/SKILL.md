# 目的
要件定義の結果をもとに、GitHub 上に親Issueを作成し、必要に応じて子Issueと作業ブランチを作成する。

# このskillがやること
- 要件定義結果を読む
- 親Issueを作成する
- 必要に応じて子Issueを作成する
- 対応する作業ブランチを作成する
- 作成結果を返す

# このskillがやらないこと
- 実装しない
- 要件にない機能を追加しない
- 過剰な設計を行わない
- 不要な説明を書かない

# 入力
要件定義結果のみを対象とする。
相談履歴は使用しない。

含まれる想定:
- 要件サマリ
- 未確定事項
- 実装案
- Issue草案

# 前提
- GitHub 操作用 tool が利用可能であること
- 対象リポジトリが明確であること
- tool が使えない場合は Issue本文案のみを出力する

# 判断ルール
1. 入力から達成したい機能を特定する
2. 親Issueとして定義する
3. 親Issueには全体の目的と完了条件を書く
4. 原則として子Issueは分割しない
5. 1PRで完結しない場合のみ子Issueに分割する
6. 子Issueは単独で実装・検証可能にする
7. 曖昧な要件は未確定事項に残す
8. 要件に存在しない内容は追加しない

# 分割ルール
以下の場合のみ分割する:
- 1PRで完結しない規模
- 明確に独立した変更
- 並列作業が必要

# ブランチルール
- ブランチの作成元、命名、PR の向きは `docs/agents/branch-rules.md` に従う

# ブランチ命名ルール
- 親Issue: `issue-<番号>`
- 子Issue: `issue-<番号>`

# 実行ルール
1. 親Issue本文を作成する
2. GitHub tool で親Issueを作成する
3. 親Issue番号を取得する
4. 親Issue用ブランチを `docs/agents/branch-rules.md` に従って作成する
5. 必要な場合のみ子Issueを作成する
6. 子Issueには親Issue番号を記載する
7. 子Issue用ブランチを `docs/agents/branch-rules.md` に従って作成する
8. すべての結果をまとめて返す

# 出力ルール
- 余計な説明は禁止
- 曖昧表現は禁止
- 要件外の内容は禁止
- 不明点は未確定事項に残す
- Issue未作成の場合は理由を明示する
